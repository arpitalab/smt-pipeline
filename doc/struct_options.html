Struct Options

Every analysis method in TrajectoryCollection accepts either the traditional MATLAB name-value syntax or a single options struct. The struct form lets you define parameters once, share them across conditions, and keep your scripts readable.

Syntax

Instead of:

tc.getFBMParameters('TrackType','relative', 'ExposureFraction',0.05, ...
                    'SubtrackLength',40, 'MinSubtrackLength',40, ...
                    'MinMeanSqStep',0.0064, 'CIMethod','profile+mcmc');

Write:

fbm_opts = struct( ...
    'TrackType',          'relative',   ...
    'ExposureFraction',    0.05,        ...
    'SubtrackLength',      40,          ...
    'MinSubtrackLength',   40,          ...
    'MinMeanSqStep',       0.0064,      ...
    'CIMethod',           'profile+mcmc');

fbm = tc.getFBMParameters(fbm_opts);
Implementation: passing a single struct as the only argument is internally equivalent to expanding it to name-value pairs. Any field name that is not a recognised parameter will produce the same error as an unrecognised name-value key.

Sharing options across conditions

Define a base options struct, then create condition-specific variants by overwriting individual fields: 

% ── Base options (shared physical parameters) ─────────────────────────
sigma_fixed = 0.040;

base_msd = struct( ...
    'TrackType',     'relative',       ...
    'MinLength',      40,              ...
    'MinMeanSqStep',  4*sigma_fixed^2, ...
    'MaxLag',         40,             ...
    'NumBootstrap',   100,            ...
    'ExposureTime',   0.01);

base_fbm = struct( ...
    'TrackType',          'relative',        ...
    'ExposureFraction',    0.05,             ...
    'SubtrackLength',      40,               ...
    'MinSubtrackLength',   40,               ...
    'MinMeanSqStep',       4*sigma_fixed^2,  ...
    'CIMethod',           'profile+mcmc');

% ── Condition A ────────────────────────────────────────────────────────
opts_A      = base_msd;
opts_A.Condition = 'Control';
tc_A.recomputeMSD(opts_A);
fbm_A = tc_A.getFBMParameters(setfield(base_fbm,'Condition','Control'));

% ── Condition B (same parameters, different object) ────────────────────
tc_B.recomputeMSD(base_msd);
fbm_B = tc_B.getFBMParameters(base_fbm);

Modifying a single field

Use setfield or direct assignment to create a modified copy without changing the original: 

% Test with a stricter immobility threshold
strict_opts       = base_fbm;
strict_opts.MinMeanSqStep = 4 * 0.050^2;   % tighter threshold

fbm_strict = tc.getFBMParameters(strict_opts);

Saving and loading option sets

Structs can be saved to a .mat file alongside results for full reproducibility: 

save('results/ER_100k_slow.mat', 'fbm', 'msd_results', 'base_fbm', 'base_msd');
