fBM Analysis

The pipeline fits fractional Brownian motion (fBM) to 2D single-molecule tracks using two complementary methods: an ensemble Toeplitz maximum-likelihood estimator (MLE) and a bootstrap ensemble MSD least-squares fit. This page covers getFBMParameters, plotFBMFit, and how to interpret the relationship between the two methods.

Background: the Backlund exposure-time model

Both methods use the Backlund et al. (Phys. Rev. E, 2015) covariance kernel, which accounts for the finite camera exposure time Te: 

ψ(τ, Tₑ, α) = [(τ+Tₑ)^(α+2) + |τ−Tₑ|^(α+2) − 2τ^(α+2)]
               / [Tₑ² (α+1)(α+2)]

The expected 2D ensemble MSD at lag n frames is: 

MSD₂ᴅ(n) = 4K [ψ(n·dt, Tₑ, α) − ψ(0, Tₑ, α)] + 4σ²

where K is the per-axis generalized diffusion coefficient (µm²/sα), α is the anomalous exponent (0<α<2), and σ is the per-axis localization noise (µm). Note that ψ(0) = 2Teα/[(α+1)(α+2)] is non-zero when the exposure time is finite.

Which K? K in this pipeline is always the per-axis (1D) generalized diffusion coefficient. The 2D MSD scales as 4K, matching the standard convention MSD = 4Dt for Brownian motion (α=1).

getFBMParameters

Fits K, α, σ by maximising the Toeplitz Gaussian log-likelihood of the 2D displacement time series. x and y are treated as independent with identical covariance: 

ℓ = Σ_L N_L·(−n·log 2π − log|Σ_L|) − ½ tr(Σ_L⁻¹ S_L)

where SL = Σk(ΔxkΔxkᵀ + ΔykΔykᵀ) is the sufficient-statistic matrix for subtracks of length L, computed once at startup for O(n³) per-evaluation cost.

Signature

results = tc.getFBMParameters(opts) results = tc.getFBMParameters('TrackType','relative', 'ExposureFraction',0.05, ...)

Parameters

NameDefaultDescription
TrackType'culled' 'culled' | 'relative' | 'raw'. Use 'relative' for cell-motion-corrected tracks.
ExposureFraction1 Te/dt ratio. Set to exposure_time / frame_interval (e.g. 0.05 for 10 ms exposure at 200 ms interval). Default 1 = stroboscopic.
SubtrackLength20 Target frames per subtrack. Controls the maximum lag used in the covariance fit. Longer subtracks give more information per track but require longer tracks.
MinSubtrackLength10 Minimum frames for a subtrack to be included. Setting MinSubtrackLength = SubtrackLength uses only complete subtracks of a single fixed length.
MinMeanSqStep0 Immobility threshold (µm²). Tracks with mean one-step squared displacement below this value are excluded. Typical value: 4·σfixed² where σfixed is measured from immobile particles. See Track Filtering.
CIMethod'profile' Confidence interval method: 'profile' (likelihood-ratio, recommended), 'profile+mcmc' (profile-calibrated MCMC), 'mcmc', 'hessian' (fast but approximate), 'none'.
ProfilePoints60 Grid points per parameter for profile CI.
MCMCSamples5000 Posterior samples after burn-in (MCMC methods only).
Condition[] Restrict to one condition string. Results from subset queries are returned but not cached on the object.
ForceRecomputefalse Bypass cache and refit. The cache is automatically invalidated when any of ExposureFraction, SubtrackLength, MinSubtrackLength, MinMeanSqStep, or CIMethod changes.

Example

sigma_fixed = 0.040;   % µm, from fixed-particle measurement

fbm_opts = struct( ...
    'TrackType',        'relative',       ...
    'ExposureFraction', 0.05,             ...
    'SubtrackLength',   40,               ...
    'MinSubtrackLength',40,               ...
    'MinMeanSqStep',    4*sigma_fixed^2,  ...
    'CIMethod',         'profile+mcmc');

fbm = tc.getFBMParameters(fbm_opts);

fprintf('alpha = %.3f  [%.3f, %.3f]\n', ...
        fbm.alpha, fbm.CI.alpha(1), fbm.CI.alpha(2));
fprintf('K     = %.4g  [%.4g, %.4g] µm²/s^alpha\n', ...
        fbm.K, fbm.CI.K(1), fbm.CI.K(2));
fprintf('sigma = %.4f  [%.4f, %.4f] µm\n', ...
        fbm.sigma, fbm.CI.sigma(1), fbm.CI.sigma(2));

plotFBMFit

Overlays the Toeplitz MLE model curve on the empirical ensemble MSD. Requires getMSD to have been run first. Because getFBMParameters with TrackType='relative' does not cache results on the object, the FBM result struct must be passed explicitly via the FBMResults parameter.

Signature

figs = tc.plotFBMFit('FBMResults', fbm) figs = tc.plotFBMFit(struct('FBMResults',fbm, 'Title','ER 100k Slow'))

Parameters

NameDefaultDescription
FBMResults[] Struct returned by getFBMParameters. Required when using TrackType='relative' or a Condition filter; optional for culled-track results cached on the object.
Color[0 0.4 0.8] RGB colour for the empirical MSD and CI shading.
Titleauto Figure title. Defaults to "MSD + Toeplitz MLE (n=N tracks)".
ShowBootstraptrue Overlay the median bootstrap MSD least-squares fit curve for comparison with the Toeplitz MLE curve.

What the plot shows

ElementDescription
Shaded band95% bootstrap CI on the ensemble mean MSD.
Solid lineEmpirical ensemble mean TAMSD (time-and-ensemble average).
Dashed coloured lineMedian bootstrap MSD least-squares fit (shown when ShowBootstrap=true). Reports α from the MSD fit.
Dashed black lineToeplitz MLE model curve computed from K, α, σ using the exact Backlund formula. Reports MLE α, K, σ.

Interpreting differences between the two curves

ObservationLikely cause
MLE curve matches MSD, same αData is consistent with pure fBM. Both estimators agree.
MLE α > MSD α, curves diverge at short lags MSD fit is biased by the localization noise floor at short lags, making the slope appear shallower. MLE α is more reliable.
Parallel offset (same α, MLE curve lower) MSD ensemble includes shorter tracks with higher localization noise than the subpopulation used by the MLE. Apply MinLength to align the two populations.
Large discrepancy, curves cross Data is likely not pure fBM — mixed states, confinement, or non-stationary dynamics.

Full workflow example

sigma_fixed = 0.040;   % µm, independently measured

% --- Options structs ---
msd_opts = struct( ...
    'TrackType',     'relative', ...
    'MinLength',      40,         ...
    'MinMeanSqStep',  4*sigma_fixed^2, ...
    'MaxLag',         40,         ...
    'NumBootstrap',   100,        ...
    'ExposureTime',   0.01);

fbm_opts = struct( ...
    'TrackType',          'relative',        ...
    'ExposureFraction',    0.05,             ...
    'SubtrackLength',      40,               ...
    'MinSubtrackLength',   40,               ...
    'MinMeanSqStep',       4*sigma_fixed^2,  ...
    'CIMethod',           'profile+mcmc');

% --- Compute ---
tc.recomputeMSD(msd_opts);
fbm = tc.getFBMParameters(fbm_opts);

% --- Visualise ---
tc.plotFBMFit(struct('FBMResults', fbm, 'Title', 'ER 100k Slow'));

Choosing SubtrackLength

SubtrackLength controls how many consecutive displacement increments enter each covariance matrix. Longer subtracks capture more of the fBM correlation structure, but require longer tracks and increase compute time.

SubtrackLengthNotes
10–15Fast. Suitable for noisy data or when tracks are short. Less information about long-lag correlations.
20Good default for most SMT experiments.
40Recommended when tracks are long and you want to resolve the full correlation structure. Also aligns the MLE track population with a MinLength=40 MSD filter.
Tip: set SubtrackLength = MinSubtrackLength to use only complete subtracks of a single fixed length. This gives a homogeneous covariance matrix size and avoids any averaging artefacts from mixing subtrack lengths.

getFBMAlphaDistribution

Fits (K, α) independently to each track to obtain the full per-track distribution of anomalous exponents. Useful for characterising heterogeneity in the sample and comparing state fractions to RL or pEM classifications.

results = tc.getFBMAlphaDistribution() results = tc.getFBMAlphaDistribution('TrackType','relative', 'FixSigma',true, ... 'MinTrackLength',20, 'MinStepVar',4*sigma^2)

Parameters

ParameterDefaultDescription
TrackType'culled''culled' | 'relative' | 'raw'
FixSigmatrueFix σ to the population MLE value (from getFBMParameters), reducing per-track fitting from 3D to 2D and substantially reducing noise in α estimates.
SigmaValueNaNOverride σ (µm) directly instead of using the auto-estimated population value.
ExposureFraction1Te/dt. Should match the value used in getFBMParameters.
MinTrackLength15Minimum frames per track to include.
MaxSubtrackLength0Cap subtrack length for per-track fits. Set to match SubtrackLength from getFBMParameters (e.g. 40) for consistency. Default 0 = use full track.
MinStepVar0Pre-filter: drop tracks with mean squared step < threshold (µm²). Use 4·σ² to exclude stuck particles.
MinAlpha0Post-fit floor: tracks with fitted α < MinAlpha are removed from results.
Condition[]Restrict to a condition string.
ForceRecomputefalseRerun even if cached.
VerbosefalsePrint per-track fitting progress.

Result fields

FieldDescription
.alphaN×1 per-track anomalous exponent.
.KN×1 per-track K (µm²/sα).
.KaN×1 per-track K·α.
.track_lengthN×1 length (frames) of each track.
.sigma_fixedScalar σ used (NaN if fitted jointly).

Interpretation caveat: With ~38 scalar observations per 20-frame track, per-track α estimates have intrinsic variance of ~0.2–0.4 s.d. The observed distribution of α is a convolution of the true biological distribution with this fitting noise. Do not over-interpret narrow features in the α histogram.

Example

% Run population MLE first to get sigma
fbm = tc.getFBMParameters('TrackType','relative', ...
    'ExposureFraction',0.05,'SubtrackLength',40,'CIMethod','none');

% Per-track distribution with sigma fixed to population value
alpha_dist = tc.getFBMAlphaDistribution( ...
    'TrackType',        'relative', ...
    'ExposureFraction',  0.05,      ...
    'MaxSubtrackLength', 40,        ...
    'FixSigma',          true,      ...
    'MinTrackLength',    20,        ...
    'MinStepVar',        4*fbm.sigma^2);

figure;
histogram(alpha_dist.alpha, 30, 'Normalization','probability');
xlabel('\alpha'); ylabel('Fraction');
title(sprintf('Per-track \\alpha  (n=%d, \\sigma=%.3f µm)', ...
    alpha_dist.n_tracks, alpha_dist.sigma_fixed));

Confidence intervals

Three CI methods are available:

MethodSpeedWhen to use
'profile'Minutes Default. Likelihood-ratio intervals; correct for asymmetry and banana-shaped posteriors. Sweeps a grid of 60 points per parameter (adjustable via ProfilePoints).
'profile+mcmc'Minutes–hours Profile first, then short MCMC (300-step burn-in) calibrated by the profile marginals. Most accurate; use for final reported values.
'mcmc'Hours Full adaptive Metropolis-Hastings from the MLE. Use when the profile is unreliable (very flat likelihood).
'hessian'Seconds Laplace approximation. Fast but poor for this problem; diagnostic use only.
'none'Seconds MLE only, no CI. Useful for quick exploration.
Why CI is slow when SubtrackLength ≠ MinSubtrackLength: Each profile step must invert a separate covariance matrix for every distinct subtrack length present in the data. When all subtracks have the same length (SubtrackLength = MinSubtrackLength), only one matrix size is needed and CI computation is much faster.
