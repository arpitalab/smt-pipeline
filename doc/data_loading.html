Data Loading

The pipeline has two classes responsible for loading and organising track data: TrajectoryWrapper (one per FOV) and TrajectoryCollection (aggregates all FOVs and owns all analysis methods).

TrajectoryWrapper

TrajectoryWrapper is a handle class that owns the raw data for a single field of view. It handles:

  • Loading from CSV (Python quot tracker output) or from a processed SMD object.
  • Converting coordinates from pixels to µm.
  • Culling tracks using a nucleus/ROI mask and a gap-consistency filter.
  • Removing global cell motion via a maximum spanning forest.

Track storage format

After cull_tracks(), each track in RawTracks is an N×5 matrix:

ColumnContents
1x (µm)
2y (µm)
3frame (1-based integer)
4SNR (per-localization)
5ROI_ID (> 0 = passed culling; −1 = rejected)

getCulledTracks() returns only the entries where col 5 > 0.

CSV workflow

tw = TrajectoryWrapper('PixelSize', 0.165, 'FrameInterval', 0.02);
tw.readData('s001_trajs.csv');      % loads, converts px→µm, 0-based→1-based frames
tw.loadNucleusMask();               % finds matching CellPose .mat mask
tw.cull_tracks();                   % ROI membership + gap filter
tw.remove_relative_motion();        % maximum spanning forest subtraction

tracks = tw.getCulledTracks();      % N×5 cell array, culled
rel    = tw.getRelativeTracks();    % relative-motion-corrected tracks
tw.summary();                       % prints track counts, pixel size, dt
CSV format: The reader expects columns in the order produced by the quot Python tracker: x, y, frame, SNR, track_id (columns 1, 2, 16, 11, 18 of the CSV). Coordinates are in pixels and are scaled by PixelSize on load. Frames are shifted from 0-based (Python) to 1-based (MATLAB).

Reading parameters from a TOML file

tw.readParams('experiment.toml');
% Recognized keys:
%   pixel_size     → PixelSize (µm/px)
%   frame_interval → FrameInterval (s)
% All other keys are stored in tw.Parameters

A minimal TOML file:

[acquisition]
pixel_size     = 0.165
frame_interval = 0.02

[culling]
minTrackLength    = 5
minLengthBeforeGap = 3

Culling logic

cull_tracks() calls TrackUtils.cullTracksCore(), which:

  1. Tests whether each track's starting position falls inside any ROI polygon.
  2. Applies a gap-consistency filter: keeps only frames before the first gap longer than minLengthBeforeGap frames.
  3. Trims each passing track to its valid start frame.

Tracks that fail are kept in RawTracks with col 5 = −1 (so the full track list is preserved for auditing); getCulledTracks() filters them out.

Relative motion removal

remove_relative_motion() corrects for global nuclear drift within each ROI:

  1. Groups culled tracks by ROI_ID.
  2. Builds a maximum spanning forest connecting co-temporal tracks (requires ≥ 10 tracks per ROI).
  3. Computes pairwise relative displacements along forest edges to estimate the global drift at each frame.
  4. Subtracts the estimated drift from every track; result stored in RelativeTracks.

Results are available via tw.getRelativeTracks().

SMD workflow

% The SMD object must already be processed:
smd.localize();
smd.track();
smd.get_roi();    % optional — if absent, all tracks get ROI_ID = 1
smd.cull_tracks();

tw = TrajectoryWrapper.fromSMD(smd);
% Imports tracks already in µm (smdTracksToMicrons), sets FrameInterval
% from smd.frame_rate, copies ROIs and relative tracks if present.

TrajectoryCollection

TrajectoryCollection aggregates any number of TrajectoryWrapper instances and owns all analysis methods (MSD, fBM, RL, pEM, Lifetime). It maintains a Metadata table (FileID, FilePath, Condition) and lazy-collected track arrays per type.

addFromFile

tc.addFromFile(csvFile)
tc.addFromFile(csvFile, 'FileID','1', 'Condition','Control', ...
               'PixelSize',0.165, 'FrameInterval',0.02)
tc.addFromFile(csvFile, 'ParamsFile','experiment.toml', 'Condition','Treated')

Internally creates a TrajectoryWrapper, runs readData, loadNucleusMask, cull_tracks, and remove_relative_motion, then registers it in the collection. Direct PixelSize/FrameInterval arguments override TOML values.

addFromSMD

tc.addFromSMD(smd)
tc.addFromSMD(smd, 'FileID','run2', 'Condition','Treated')

Creates a TrajectoryWrapper via TrajectoryWrapper.fromSMD(smd) and registers it. No pipeline steps are re-run; the SMD must be fully processed.

ReadParams — propagate TOML to all wrappers

tc.ReadParams('experiment.toml')
% Sets PixelSize on CSV-sourced wrappers and FrameInterval on any
% wrapper where it is still NaN. Stores all keys in tc.Params.

setFrameIntervalAll

tc.setFrameIntervalAll(0.02)   % override FrameInterval on every wrapper

Batch loading

files = dir('data/*.csv');
for k = 1:numel(files)
    tc.addFromFile(fullfile(files(k).folder, files(k).name), ...
                   'FileID',    num2str(k), ...
                   'Condition', 'Control');
end
tc.summary();

Querying

% By condition
tracks = tc.getTracksByCondition('Control');
rel    = tc.getRelativeTracksByCondition('Control');

% All tracks (lazy-collected across all wrappers)
all_culled   = tc.getAllCulledTracks();
all_relative = tc.getAllRelativeTracks();

% Track length distribution (culled)
tl = tc.getTrackLengthDistribution();
histogram(tl, 50);

% Visualization: plot tracks per wrapper
tc.visualizeSubsets('Condition','Control','NumTracks',200);

Conditions

Any string can be used as a Condition tag. All analysis methods accept a 'Condition' argument to restrict the track population. When Condition is set, results are not cached on the object (they are returned but not stored as the primary cached result). 

% Set condition after loading
tc.setCondition('3', 'Treated');   % change condition for FileID '3'
