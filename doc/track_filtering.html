Track Filtering

Two optional filters can be applied in getMSD and getFBMParameters before any computation: a minimum track length filter and an immobility filter. Both are applied to the track list after loading and before any MSD or likelihood computation.

MinLength — minimum track length

Excludes tracks shorter than a given number of frames. Applied to both getMSD and getFBMParameters.

Why use it

The Toeplitz MLE requires subtracks of at least MinSubtrackLength frames. Short tracks that fail this criterion contribute nothing to the MLE but do contribute to the ensemble MSD — inflating the noise floor with poorly localised molecules. Matching the MinLength in getMSD to the SubtrackLength / MinSubtrackLength used in getFBMParameters ensures both methods see the same track population, eliminating systematic vertical offsets in plotFBMFit.

Usage

% getMSD: include only tracks >= 40 frames
tc.recomputeMSD('TrackType','relative', 'MinLength',40, ...
                'MaxLag',40, 'ExposureTime',0.01);

% getFBMParameters: same population
fbm = tc.getFBMParameters('TrackType','relative', ...
                           'SubtrackLength',40, 'MinSubtrackLength',40, ...);
Rule of thumb: set MinLength = SubtrackLength so the MSD and MLE see exactly the same tracks.

MinMeanSqStep — immobility filter

Excludes tracks whose mean one-step squared displacement (µm²) falls below a threshold. Applied to both getMSD and getFBMParameters. 

mean_sq_step(track) = mean( diff(x).^2 + diff(y).^2 )

Why remove immobile tracks

Immobile particles (stuck to the coverslip, aggregated, or non-specifically bound) appear as pure localization noise in the fBM model. Their inclusion:

  • Inflates the estimated σ, pulling it toward the localization noise of stationary emitters rather than diffusing ones.
  • Pulls α toward 0 — immobile tracks look like extremely subdiffusive fBM.
  • Inflates the short-lag MSD, making the power-law slope appear shallower.

Choosing the threshold

The recommended threshold is 4·σfixed², where σfixed is the per-axis localization precision measured from truly immobile particles (e.g. fixed fluorescent beads or adsorbed single molecules) imaged under identical conditions. 

% From fixed-particle measurement
sigma_fixed = 0.040;            % µm  (40 nm)
thresh = 4 * sigma_fixed^2;     % µm² (= 0.0064 µm²)

% Apply in both methods
tc.recomputeMSD('MinMeanSqStep', thresh, ...);
fbm = tc.getFBMParameters('MinMeanSqStep', thresh, ...);
Why 4σ²? The expected 2D mean squared step for a truly immobile particle is 4σ² (the sum of two independent noise contributions, one per displacement, each of variance 2σ² across two axes). Any track below this threshold has less average displacement than you would expect from localization noise alone, so it cannot be distinguished from an immobile particle.
Relation to SNR filtering: If your localizer already rejects individual spots below a minimum SNR (e.g. SNR < 2), low-quality frames are excluded at the track level — tracks end or acquire gaps when SNR drops. The MinMeanSqStep filter operates at the track level and is complementary: it removes tracks that are systematically non-mobile regardless of per-frame SNR.

Console output

When the filter removes tracks, a summary is printed:

getMSD: removed 312 immobile tracks (MinMeanSqStep=0.0064 µm²), 4105 remain.
getFBMParameters: removed 312 immobile tracks (MinMeanSqStep=0.0064 µm²), 4105 remain.

Filter order

Filters are applied in the following order inside each method:

  1. Track loading — all tracks of the requested TrackType and Condition are loaded.
  2. MinLength — short tracks are dropped.
  3. MinMeanSqStep — immobile tracks are dropped.
  4. Computation — MSD or MLE runs on the surviving tracks.

Combined example

sigma_fixed = 0.040;   % µm

msd_opts = struct( ...
    'TrackType',     'relative',      ...
    'MinLength',      40,             ...
    'MinMeanSqStep',  4*sigma_fixed^2,...
    'MaxLag',         40,            ...
    'NumBootstrap',   100,           ...
    'ExposureTime',   0.01);

fbm_opts = struct( ...
    'TrackType',          'relative',        ...
    'MinMeanSqStep',       4*sigma_fixed^2,  ...
    'ExposureFraction',    0.05,             ...
    'SubtrackLength',      40,               ...
    'MinSubtrackLength',   40,               ...
    'CIMethod',           'profile+mcmc');

tc.recomputeMSD(msd_opts);
fbm = tc.getFBMParameters(fbm_opts);
tc.plotFBMFit(struct('FBMResults', fbm, 'Title', 'ER 100k Slow'));

With matching MinLength and MinMeanSqStep, the empirical MSD and the Toeplitz MLE curve in plotFBMFit will reflect the same track population. Any remaining vertical offset between the curves reflects a genuine difference in what the two estimators extract from the data, not a population mismatch.
